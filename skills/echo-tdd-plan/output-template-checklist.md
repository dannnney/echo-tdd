## 验证层次说明

环境前置条件按依赖关系分为 6 层，必须按顺序验证：

```
第 1 层：基础运行环境
  └─ 语言版本、包管理器、基础工具
  └─ 如果失败 → 后续所有层都无法验证

第 2 层：依赖安装
  └─ 测试框架、SDK、工具库
  └─ 依赖：第 1 层通过
  └─ 验证方式：尝试 import/require

第 3 层：认证/凭证
  └─ 凭证配置、凭证有效性
  └─ 依赖：第 2 层通过
  └─ 如果失败 → 所有需要认证的通道都无法验证

第 4 层：基础通道可达性
  └─ DB 连通性、API 可达性、浏览器可启动
  └─ 依赖：第 1-3 层通过
  └─ 这是核心层，决定哪些观测通道可用

第 5 层：组合通道可用性
  └─ SDK 功能、辅助脚本
  └─ 依赖：第 3-4 层通过
  └─ 这一层失败可以降级，不是致命的

第 6 层：数据操作权限
  └─ 创建、读取、删除测试数据的权限
  └─ 依赖：第 4-5 层通过
  └─ 这一层决定数据准备和清理策略
```

**探测原则**：
- 按顺序逐层验证，不跳过
- 某层失败时，询问用户是否跳过依赖此层的后续验证
- 记录每项的 PASS/FAIL/WARN 状态

**示例探测脚本**：
```javascript
// probe-browser.js
import { chromium } from '@playwright/test';

async function probeBrowser() {
  const browser = await chromium.launch({ headless: true });
  const page = await browser.newPage();
  await page.goto('http://localhost:3000');
  await page.waitForLoadState('domcontentloaded');
  const title = await page.title();
  await browser.close();
  console.log(`✅ Browser probe passed: ${title}`);
}

probeBrowser().catch(err => {
  console.error(`❌ Browser probe failed: ${err.message}`);
  process.exit(1);
});
```

## 后续使用

此 checklist 将在阶段二（环境探测验证）中按顺序逐项执行。每项探测结果记录为：

- **✅ PASS**：验证通过，该项环境就绪
- **❌ FAIL**：验证失败，需要修复或寻找替代方案
- **⚠️  WARN**：验证部分通过，有警告但不影响核心功能

## 注意事项

1. **按顺序验证**：必须从第 1 层开始，不要跳过
2. **某层失败时的处理**：
   - 关键层（1-4）失败 → 询问用户是否修复后重试
   - 非关键层（5-6）失败 → 记录为 WARN/FAIL，调整策略继续
3. **动态调整**：根据探测结果，阶段二会调整可观测性方案
4. **用户确认**：探测可能涉及创建/删除数据，执行前需用户确认
