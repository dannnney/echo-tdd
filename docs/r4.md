"可观测性” 就是指我们在测试时，模拟人类测试人员去观察系统是否测试达标的一种理念。

比如测试工程师在测试时，他们可以基于不同的环境、不同的基建可达程度从而有选择的进行验证。

我把可观测性分为以下几种场景：
1. 基于浏览器页面进行观测
2. 基于APP/客户端界面进行观测
3. 基于浏览器控制台进行观测
4. 基于 curl/浏览器 对 HTTP API 的 Get、Post 等 （包括但不限于 GraphQL 等）进行观测
5. 基于 cli 命令行的返回值进行观测
6. 对已经存在的 SDK 编写代码，运行脚本的输出进行观测
7. 从日志（日志系统、本地日志文件等）进行观测
8. shell stdin/stdout 进行观测
9. 其他


我认为不同的测试人员、环境、权限下，人类会基于自己的现实条件进行测试以及验证，比如：

1. 比如在浏览器触发界面，然后用 postgres cli 读取数据结果进行观测
2. 比如在浏览器触发界面，然后在浏览器控制进行网络观测
3. 比如在 Postman/curl/浏览器 触发 HTTP 请求，然后在 UI 界面进行结果观测
4. 假设后台开发者没有数据库直接访问的权限，但是它可以开发一个数据提取的 API ，所以可以发起一个 Post 请求（业务需求），然后再 Get （临时开发用于测试的辅助 API ） 出来验证。
5. 如果涉及到多角色，比如可以在 UI 界面用员工身份发起审批请求，然后用管理员角色在 UI 界面观察是否收到这个审批请求
6. 等等


所以这就是我的可观测性的基础理念
有了这个理念，我们就可以根据用户的访谈，来最终确定我们的可观测性的解决方案提案，供用户确认
